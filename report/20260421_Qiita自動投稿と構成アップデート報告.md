# 作業報告書: Qiita への自動投稿の仕組み作成と構成のアップデート

**作業日**: 2026年4月21日

## 概要

Qiita CLI を導入し、ローカルの Markdown ファイルから Qiita への投稿・更新を自動化する仕組みを構築した。併せて、リポジトリ全体の構成を Qiita 統一に変更した。

## 1. Qiita CLI の導入

### インストール

- `@qiita/qiita-cli` を devDependencies にインストール
- `npx qiita init` で初期設定ファイルを生成
- `npx qiita login` で Qiita API トークンを使って認証

### 主要コマンド

| コマンド | 用途 |
|---|---|
| `npx qiita preview` | ブラウザでプレビュー |
| `npx qiita new <basename>` | 新規記事の作成 |
| `npx qiita publish <basename>` | 指定記事を投稿・更新 |
| `npx qiita publish --all` | 全記事を投稿・更新 |
| `npx qiita pull` | Qiita から記事を同期 |

### 追加されたファイル

| ファイル | 説明 |
|---|---|
| `package.json` | Node.js プロジェクト設定 |
| `package-lock.json` | 依存関係のロックファイル |
| `qiita.config.json` | Qiita CLI の設定ファイル |

## 2. GitHub Actions による自動投稿

### ワークフロー

`.github/workflows/publish.yml` を作成。`main` ブランチへの push 時に `public/` 配下に変更があれば、自動的に Qiita へ記事を投稿・更新する。

```yaml
on:
  push:
    branches:
      - main
    paths:
      - "public/**"
  workflow_dispatch:
```

### 必要な設定

- GitHub リポジトリの Settings → Secrets and variables → Actions で `QIITA_TOKEN` を登録済み

## 4. 画像の表示対応

### リポジトリの公開化

リポジトリを private → public に変更。これにより GitHub raw URL で画像を直接参照可能になった。
